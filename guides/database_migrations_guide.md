# Database Migrations Guide (Alembic)

This document provides a complete production-grade workflow for handling incremental database schema changes (migrations) using **Alembic** in the Structural Design Copilot API.

---

## 1. Quick Concept

* **`init_db.py`**: Best for a clean database boot or tests. It runs `create_all()`, which creates missing tables but **cannot** alter or modify existing tables when their code schemas change.
* **Alembic**: The industry-standard SQLAlchemy tool that generates incremental version files (e.g. `001_add_roles.py`). It compares your local ORM models with your live database schema and writes the appropriate `ALTER TABLE` SQL commands.

---

## 2. Installation

First, ensure that `alembic` is installed in your python virtual environment:

```bash
# From the project root, run:
./venv/bin/pip install alembic
```

---

## 3. Initializing Alembic

Inside your API folder (`apps/api`), initialize the migrations environment:

```bash
cd apps/api
../../venv/bin/alembic init -t async migrations
```
> [!NOTE]
> The `-t async` flag is **critical**. It initializes Alembic with a template preconfigured for asynchronous drivers like `asyncpg`.

This command creates:
1. `alembic.ini`: The central configuration file.
2. `migrations/`: A directory containing migration logic, versions, and the `env.py` engine setup file.

---

## 4. Configuration

To wire Alembic to automatically discover your database URL and ORM models, make these two quick updates:

### File 1: `apps/api/alembic.ini`
Look for the `sqlalchemy.url` line and leave it empty or generic because we will load it dynamically from our `config.py` environment variables instead:
```ini
# Leave this line blank (we override it dynamically in env.py)
sqlalchemy.url = 
```

### File 2: `apps/api/migrations/env.py`
Open `migrations/env.py`. Locate the imports section and configure it to import your Pydantic settings and ORM `Base`:

```python
# 1. Add these imports near the top of migrations/env.py
import sys
import os
import re

# Append current apps/api directory to path so python can discover modules
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

from config import settings
from db.base import Base
import db.models  # Ensures all ORM models are registered on Base.metadata

# 2. Update the target_metadata variable:
target_metadata = Base.metadata

# 3. Locate the run_migrations_online() function and update the connection setup:
# In an async template, locate the section where the database URL is read, and replace it:
def run_migrations_online() -> None:
    # Dynamically inject your database environment URL and map postgresql: to postgresql+asyncpg:
    db_url = re.sub(r'^postgresql:', 'postgresql+asyncpg:', settings.DATABASE_URL)
    
    # Strip any unsupported parameters (just like in session.py)
    if "?" in db_url:
        base_url, query_str = db_url.split("?", 1)
        unsupported_params = ["sslmode", "channel_binding"]
        params = []
        for param in query_str.split("&"):
            if not any(param.startswith(unsupported + "=") for unsupported in unsupported_params):
                params.append(param)
        db_url = f"{base_url}?{'&'.join(params)}" if params else base_url

    configuration = config.get_section(config.config_ini_section)
    configuration["sqlalchemy.url"] = db_url
    
    # For SSL support (e.g. Neon), pass the connection args:
    connect_args = {}
    if "sslmode" in settings.DATABASE_URL or "ssl=require" in settings.DATABASE_URL:
        connect_args["ssl"] = True

    # (Update your create_async_engine call inside run_migrations_online to pass connect_args=connect_args)
```

## 5. Standard Development Workflow (How to make schema changes)

Once Alembic is configured, follow these three steps whenever you want to change your database structures:

### Step 1: Update your ORM models
For example, if you want to add a `phone_number` string column to `users` in `apps/api/db/models/user.py`:
```python
phone_number: Mapped[Optional[str]] = mapped_column(String(50), nullable=True)
```

### Step 2: Generate an Autogenerated Migration script
Run the revision generator from your `apps/api` folder:
```bash
# Generates a version file in migrations/versions/ containing alter statements
../../venv/bin/alembic revision --autogenerate -m "Add phone number to users"
```
* Alembic inspects your PostgreSQL database, compares it with `Base.metadata`, detects that `phone_number` is missing, and generates the Python scripts to create it.
* **Always review the generated script** inside `migrations/versions/` to make sure it looks correct before applying it.

### Step 3: Apply the Migration
Run the upgrade command to execute the migration on your database:
```bash
../../venv/bin/alembic upgrade head
```
This runs the SQL statements on Neon securely and brings your remote database in sync with your local python code!

---

## 6. Going Back in Time (Rollbacks)

If you ever make a mistake or need to revert the latest change:
```bash
# Rollback the single latest migration
../../venv/bin/alembic downgrade -1
```
